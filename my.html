저는 이렇게 작업해요.

원활한 유지보수를 위해 컴포넌트 단위별로 개발 합니다.
다른 개발자와의 협업을 생각해 주석을 꼼꼼히 달아둡니다.
협업 및 형상 관리를 위해 Github 를 사용합니다.
디자이너, 개발자와의 협업을 위해 디자인과 개발 공부를 병행하고 있습니다.
배움은 원만한 소통을 도와주며 좋은 결과물을 창출합니다. 👍

사용할 수 있는 기술은요

웹 표준을 준수한 마크업이 가능합니다.
웹 표준 인증마크를 획득한 경험이 있습니다.
디바이스 별 반응형 작업이 가능합니다.

공통 및 컴포넌트 별 CSS 작성이 가능합니다.
Styled-components, SCSS 사용이 가능합니다.
bootstrap, tailwind, mantine를 활용한 CSS 작업이 가능합니다.
git을 사용할 수 있습니다. clone, add, commit, push를 이해하고 터미널에서 사용할 수 있습니다.
svn을 사용할 수 있습니다.
Jenkins, Docker를 이용하여 프로젝트를 배포해본 경험이 있습니다.

JavaScript로 DOM을 조작할 수 있습니다.
클로저를 이해하고 프로젝트에 활용할 수 있습니다.
최신 문법(ES6+)과 비동기 처리를 이해하고 있습니다.
babel을 사용하지 않고도 브라우저 별 호환이 가능하도록 코드를 작성할 수 있습니다.

React Component 별 퍼블리싱 작업이 가능합니다.
React Hook, Context API를 활용할 수 있습니다.
TypeScript를 사용해본 경험이 있습니다.
Lerna로 의존성을 관리해본 경험이 있습니다.

PHP, ASP, JSP 환경에서 퍼블리싱 작업이 가능합니다.
그누보드를 이용하여 웹 사이트를 제작할 수 있습니다.

ASP로 CRUD를 구현할 수 있습니다.
Spring Legacy&Spring Boot로 CRUD를 구현할 수 있습니다.
Spring Legacy로 API를 만들어본 경험이 있습니다.
MVC 패턴을 이해하고 있습니다.

이 직업을 선택한 이유는요.

그림 그리는 것을 좋아하는 저의 장점인 창의성을 활용하여 사이트를 디자인 할 수 있었으며, 작업한 것을 두 눈으로 명확하게 볼 수 있다는 점이 큰 매력이었습니다.
직접 디자인하고 수많은 고민을 하며 마크업을 진행한 결과물이 누군가에게 가치를 제공하고 편하게 사용할 수 있는 사이트가 되는 것에 성취감을 느꼈습니다.

문제를 해결하는 과정에서 성장하는 것을 좋아합니다 💖
해보고 싶은 것은 해봐야한다는 생각으로 웹 사이트 인터렉티브를 위한 자바스크립트를 공부를 시작으로, 현재는 React와 Next에 호기심을 가지고 개인 프로젝트를 진행하고 있습니다.

기록하는 것을 좋아합니다.

학창시절부터 다이어리 작성하는 것을 좋아해 사소한 것 하나하나 기록 하는 것이 습관이 되었습니다.
블로그에 공부한 것이나 남기고 싶은 것을 기록합니다. 나중에 다시 꺼내서 읽어볼 수 있게 말이죠. 😁 저의 기술 블로그를 소개합니다♡
